# テスト計画: 開発秘話ページの共通UI化とテーマシステム対応

## テスト方針
- リファクタリング（UIの見た目・テーマ対応）のため、既存テストの維持を優先
- 既存テストがある場合は通過確認
- 新規テストは最小限（共通UIコンポーネントは既にテスト済み）

## テストケース

### 1. 既存テスト通過確認
- [ ] `npm run test` が全テスト通過

### 2. ビルド確認
- [ ] `npm run build` が成功（型エラーなし）

### 3. Lint確認
- [ ] `npm run lint` がエラー・warning なし

### 4. 動作確認（手動/スクリーンショット）
- [ ] 一覧ページ: 通常モードで正常表示
- [ ] 一覧ページ: クリスマスモードで正常表示
- [ ] 詳細ページ: 通常モードで正常表示
- [ ] 詳細ページ: クリスマスモードで正常表示
- [ ] 前後ナビゲーション: 正常動作
- [ ] 戻るリンク: 正常動作

## カバレッジ目標
- 既存カバレッジを下げない
- 新規テスト追加は不要（純粋なUIリファクタリング）
