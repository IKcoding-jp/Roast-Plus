# requirement.md — Issue #249

## 概要
コーヒークイズの「使い方ガイド」モーダル（HelpGuideModal）をDuolingo風のステップ型オンボーディングUIにリデザインする。

## 背景・問題
現状の `HelpGuideModal.tsx` は：
- オレンジグラデーションヘッダーが重く、モーダル内でのコントラストが強すぎる
- 絵文字カード×4をスクロールで表示する構成が情報密度過多で視認性が低い
- ゲーミフィケーション要素（FSRS定着率・XP・レベル）の魅力が視覚的に伝わっていない

## 目標
- Duolingo・Anki風のステップ型UIで、概念を1画面1コンセプトで丁寧に伝える
- アニメーションと視覚的なミニビジュアルで各機能の価値を体感させる
- プロジェクトのデザインシステム（CSS変数・`@/components/ui`）に準拠する

## 受け入れ基準

### 必須
- [ ] モーダルがステップ型（4ステップ）になっている
- [ ] ステップ切り替えのスライドアニメーションが動作する
- [ ] ドットインジケーター（4ドット）が現在ステップを示す
- [ ] 最終ステップに「はじめる」ボタンが表示される
- [ ] オレンジグラデーションヘッダーが廃止されている
- [ ] CSS変数（`bg-overlay`, `bg-spot`, `text-ink` 等）が使用されている
- [ ] `@/components/ui` の `Modal` コンポーネントを使用している
- [ ] `npm run lint && npm run build` が通る

### 任意（余裕があれば）
- [ ] 各ステップにミニビジュアル（定着率ゲージ・XPバー等）がある

## スコープ外
- QuizCard内の解説（explanation）セクションは変更しない
- ページ構成（page.tsx）は変更しない
- テスト追加は不要（ビジュアルコンポーネントのため）
