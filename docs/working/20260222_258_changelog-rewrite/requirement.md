# 要件定義: 更新履歴の非エンジニア向け全面見直し

## Issue #258

## 背景・問題

アプリの更新履歴ページ（`/changelog`）と設定ページに表示される更新履歴が、
技術者向けの専門用語で書かれており、非エンジニアのユーザー（現場スタッフ8名）には読みにくい状態。

具体的な問題：
- `SplashScreenのHydrationミスマッチエラー` → アプリ利用者には意味不明
- `husky + lint-staged によるpre-commitフック` → 開発ツールの話
- `React Hooks eslint-disable箇所の見直し` → 開発者向け内部作業
- v0.1.0〜v0.5.13（2025年11月〜12月）の初期開発履歴が一切ない
- v0.11.0〜v0.12.0の間の重要な変更（ダークモード追加など）が記録されていない

## 目標

1. **既存エントリの書き直し（v0.5.14〜v0.12.0）**
   - 技術用語 → 平易な日本語（利用者が体感できる変化を軸に）
   - 開発者専用の変更は**削除**（利用者に見えない変更は掲載しない）

2. **欠落変更の追加（v0.11.0〜v0.12.0の間）**
   git履歴に存在するが未記録の変更を v0.12.0 に追記する

3. **歴史的エントリの新規追加（v0.5.14以前）**
   git履歴から v0.1.0〜v0.5.0 の5エントリを新規作成

## 受け入れ基準

- [ ] 更新履歴の全エントリが非エンジニアに理解できる言葉で書かれている
- [ ] 開発者専用の変更（テストフレームワーク、Lint設定、リファクタリング等）が含まれていない
- [ ] v0.1.0〜v0.5.0の歴史的エントリが存在し、正確な日付・内容が記載されている
- [ ] v0.12.0に v0.11.0〜v0.12.0間の重要な変更（ダークモード等）が記録されている
- [ ] `detailed-changelog.ts` と `version-history.ts` の両方が更新されている

## 書き直しの原則

### 良い記述例
- ❌ `SplashScreenのHydrationミスマッチエラーを解消`
- ✅ `アプリ起動時に画面が一瞬ちらつく問題を修正しました`

- ❌ `Vitestテストフレームワークを導入`
- ✅ （削除：ユーザーに関係なし）

- ❌ `Firebase関数とカメラ処理のエラーハンドリングを強化`
- ✅ `カメラ読み取りのエラー表示を改善しました`

### 削除すべき種類
- テストフレームワーク（Vitest）の導入・変更
- ESLint / husky / lint-staged の設定
- コードのリファクタリング（ファイル分割、内部整理）
- デバッグ用コードの削除
- セキュリティヘッダーの設定（CORSなど）
- CI/CD、GitHub Actions関連
- 型定義・インターフェースの変更
