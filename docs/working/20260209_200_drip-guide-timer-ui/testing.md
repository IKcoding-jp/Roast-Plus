# テスト計画

## テスト戦略

### 手動テスト（メイン）

UI改善のため、目視確認と動作テストが中心。

#### 表示確認
- [ ] スマホで「今のステップ」が一目で分かるか
- [ ] スマホで「次のステップ」が一目で分かるか
- [ ] 全体進捗（何番目/全何ステップ）が表示されるか
- [ ] 水量目標が目立つか
- [ ] デスクトップでも適切に表示されるか

#### 動作確認（自動モード）
- [ ] タイマー開始・停止・リセットが正常
- [ ] ステップ自動切替が正常
- [ ] 音声案内（3秒前カウントダウン）が正常
- [ ] 完了画面が正常に表示される

#### 動作確認（手動モード）
- [ ] 前へ/次へボタンが正常
- [ ] タイマーが無限に進む（時間制限なし）
- [ ] 完了ボタンが正常

#### テーマ確認
- [ ] 通常モードで正常表示
- [ ] クリスマスモードで正常表示

### 自動テスト

- [ ] 既存テストの pass 確認（リグレッションなし）
- [ ] Lint チェック pass

## カバレッジ目標

- 既存テストの維持
- UI変更が大きい場合は、新規コンポーネントのユニットテスト追加を検討
