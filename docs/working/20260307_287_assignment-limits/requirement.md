# 要件定義

**Issue**: #287
**作成日**: 2026-03-07
**ラベル**: enhancement

## ユーザーストーリー

ユーザー「担当表の班や作業をどんどん追加してしまって、画面からはみ出してしまう。1画面に収まるように制限をかけてほしい」
アプリ「班は最大4、作業は最大8、メンバーは最大15まで。上限に達したら追加ボタンが消え、Toast通知で上限を知らせる」

## 要件一覧

### 必須要件
- [ ] 班（Team）の追加上限を4に制限
- [ ] 作業ラベル（TaskLabel）の追加上限を8に制限
- [ ] メンバー（Member）の追加上限を15に制限
- [ ] 上限到達時に追加ボタン（「+」アイコン等）を非表示にする
- [ ] 上限到達時にToast通知で「最大○件までです」と表示する
- [ ] 上限定数を1ファイルに集約して管理する

### オプション要件
- なし（将来的なユーザーカスタマイズは別Issue）

## 非機能要件
- パフォーマンス: 制限チェックは定数比較のみ。パフォーマンス影響なし
- モバイル対応: デスクトップ・モバイル両方で追加ボタン非表示を適用

## 受け入れ基準
- [ ] 班を4つ追加した後、「班を追加」ボタンが非表示になる
- [ ] 作業ラベルを8つ追加した後、追加入力行の「+」ボタンが非表示になる
- [ ] メンバーを15名追加した後、「新規メンバー追加」UIが非表示になる
- [ ] 上限到達時にToast通知が表示される
- [ ] 既存データが上限を超えている場合、追加のみ制限し既存データの表示・削除は正常に動作する
- [ ] ユニットテストで上限チェックロジックを検証
