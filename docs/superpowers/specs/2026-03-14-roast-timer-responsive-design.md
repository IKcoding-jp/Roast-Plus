# ローストタイマー レスポンシブデザイン仕様

## 概要

ローストタイマー（`app/roast-timer/`）のPC/iPad向けレスポンシブレイアウトを実装する。現在のスマホ向け縦型レイアウト（リング上部 + コントロール下部）を、768px以上の画面で左右分割型レイアウトに切り替える。

## 背景と動機

#343でローストタイマーのUIをリデザインしたが、スマホレイアウトをPC/iPadではそのまま引き伸ばしているだけで、広い画面を活用できていない。リング290px固定、下部パネル230px固定のため、iPad横画面（1024×768）では大量の空白が発生する。

### 利用状況

- **メインデバイス**: スマホ（ローストタイマーは基本的にスマホのみで使用）
- **サブデバイス**: iPad横画面（チーム全体のメインデバイス）
- **PC**: 使用しない

## デザイン仕様

### ブレークポイント

| 画面幅 | レイアウト |
|--------|-----------|
| `< 768px` | 現行スマホレイアウト（変更なし） |
| `≥ 768px` | 左右分割型レイアウト |

**iPad縦画面（768×1024）への考慮**: ちょうど768pxでブレークポイントに該当する。左右分割は成立するが、各パネルが384px幅になりやや窮屈。ただしユーザーはiPadを横画面で使用するため、縦画面時の最適化は優先度低。実装後にPlaywright MCPで768×1024のスクリーンショットも確認し、致命的な崩れがないことを検証する。

### 左右分割型レイアウト構造（768px以上）

```
┌─ ヘッダー（全幅）──────────────────────┐
│ [← 戻る]                     [⚙ 設定] │
├──────────────┬──────────────────────────┤
│              │                          │
│   左パネル   │      右パネル            │
│   (50%)      │      (50%)              │
│              │                          │
│  タイマー    │  ステート別コントロール  │
│  リング      │                          │
│              │                          │
└──────────────┴──────────────────────────┘
```

- **左右比率**: 50:50
- **左パネル**: タイマーリング（全ステート共通）
- **右パネル**: ステートに応じたコントロール
- **ヘッダー**: 現行構造を維持（FloatingNav + 設定ボタン）
- **パネル境界**: `border-right: 1px solid var(--edge)` で視覚的に分割

### ヘッダー

ヘッダー構造は**現行を維持**する。`page.tsx` の `FloatingNav`（戻るボタン）と `RoastTimer.tsx` 内の設定ボタンはそのまま。左右分割時も上部にヘッダーが全幅で表示される既存の構造を変更しない。

### パネル境界

左パネルの右端に `border-right: 1px solid var(--edge)` で視覚的に分割する。

### タイマーリング（左パネル）

現行の `TimerDisplay.tsx` はリングコンテナを `width: 290px; height: 290px` 固定で描画している。レスポンシブ化では以下のように変更する：

- **スマホ（< 768px）**: 現行のまま `width: 290px; height: 290px` 固定
- **768px以上**: コンテナを `width: 85%; max-width: 340px; aspect-ratio: 1;` に変更。具体的には `TimerDisplay.tsx` の外側divに `md:w-[85%] md:max-w-[340px] md:h-auto md:aspect-square` を追加し、固定幅スタイルを `md:` で上書きする
- **SVG viewBox**: `0 0 290 290` を維持。SVGは `width: 100%; height: 100%` でコンテナに追従
- **中央配置**: 左パネルは `display: flex; align-items: center; justify-content: center;`
- **フォントサイズ**: 時間表示 `text-[64px] md:text-[72px]`（スマホ64px、iPad 72px）
- **ラベルテキスト**: `text-[10px] md:text-[13px]`

### 右パネル

- **高さ**: `flex: 1` で親コンテナの残り高さを使用（スマホ版の `height: 230px` 固定は `md:` で解除）
- **縦方向配置**: `justify-content: center` でコントロールを垂直中央に配置
- **overflow**: `hidden`（アニメーション時のはみ出し防止）

### 右パネル：ステート別コンテンツ

#### Setup（idle）

```
重さを選択（ラベル）
┌──────┐┌──────┐┌──────┐
│ 200g ││ 300g ││ 500g │  ← 3カード横並び
│ 8:00 ││ 9:00 ││10:00 │
└──────┘└──────┘└──────┘
┌────────────────────────┐
│     ▶ スタート          │  ← アクションボタン
└────────────────────────┘
```

- パディング: 32px 40px
- カード間隔: 10px
- カード高さ: パディング 20px 12px
- スタートボタン高さ: 現行の56px（`SetupPanel.tsx`）を維持、角丸14px

#### Running（焙煎中）

```
00:03 ━━━━━━━━━━━━━━ 08:00  ← プログレスバー

┌────────────────────────┐
│ 重さ: 200g │ 設定: 08:00│  ← 情報バッジ
└────────────────────────┘

┌──────────────┐┌────────┐
│  ⏸ 一時停止  ││⏭ スキップ│  ← ボタン行
└──────────────┘└────────┘
```

#### Completed（完了）

現行の `TimerControls.tsx` のcompleted表示をそのまま使用する（テキスト・情報バッジ・リセットボタン）。現行のテキスト内容やスタイルは変更しない。

```
┌────────────────────────┐
│ 重さ: 200g │ 時間: 08:00│  ← 情報バッジ
└────────────────────────┘

┌────────────────────────┐
│     ↻ リセット          │  ← リセットボタン
└────────────────────────┘
```

### ステート遷移アニメーション

**右パネルのみフェード/スライドで切替。左パネルのリングは滑らかに色・進捗が変化する。**

#### スマホ（< 768px）
現行のFramer Motion `panelVariants` をそのまま維持（`y: 10` / `y: -10`, `duration: 0.25`）。

#### 768px以上（左右分割時）
Framer Motionの `panelVariants` をレスポンシブに分岐する。Framer Motionのvariantsは JS値のため、`useMediaQuery('(min-width: 768px)')` カスタムフックで画面幅を検知し、variants オブジェクトを切り替える：

| 遷移 | 左パネル（リング） | 右パネル |
|------|-------------------|---------|
| Setup → Running | リング色: CSS transitionで0.6s ease遷移、プログレス描画開始 | exit: `{ opacity: 0, y: -20, transition: { duration: 0.4 } }` → enter: `{ opacity: 1, y: 0, transition: { duration: 0.4 } }` |
| Running → Completed | リング色: オレンジ → グリーンに0.6s ease遷移 | 同様のフェード切替 |
| Completed → Setup | リング色: グリーン → グレーに0.6s ease遷移、プログレスリセット | 同様のフェード切替 |

リングの色遷移はCSS transition（`transition: stroke 0.6s ease`）で実現し、Framer Motionとは独立して動作する。

**重要**: 非アクティブな右パネルには `pointer-events: none` を設定し、クリックブロックを防止する。

### ボタン動作

すべてのボタンは**確認ダイアログなしで即座にステートを切り替える**。

| ボタン | 動作 | 確認 |
|--------|------|------|
| 重さカード | 選択切替 → リングの時間表示が即時更新 | なし |
| スタート | → Running へ遷移 | なし |
| 一時停止 | リング停止、ボタンが「▶ 再開」に変化 | なし |
| 再開 | リング再開、ボタンが「⏸ 一時停止」に変化 | なし |
| スキップ | → Completed へ遷移 | なし |
| リセット | → Setup へ遷移 | なし |

## 実装方針

### 変更対象ファイル

| ファイル | 変更内容 |
|---------|---------|
| `components/RoastTimer.tsx` | メインレイアウトを左右分割に変更（md以上）、useMediaQuery使用 |
| `components/roast-timer/TimerDisplay.tsx` | リングサイズのレスポンシブ化 |
| `components/roast-timer/SetupPanel.tsx` | 右パネル用スタイル追加 |
| `components/roast-timer/TimerControls.tsx` | 右パネル用スタイル追加 |
| `hooks/useMediaQuery.ts` | 新規作成（window.matchMediaベースの軽量フック） |

### レスポンシブ実装手法

- **レイアウト切替**: Tailwind `flex-col md:flex-row` でスマホ=縦、iPad=横
- **リングサイズ**: Tailwind `md:w-[85%] md:max-w-[340px] md:h-auto md:aspect-square` で制御
- **フォントサイズ**: Tailwind `text-[64px] md:text-[72px]` 等で段階的に拡大
- **アニメーション分岐**: `useMediaQuery('(min-width: 768px)')` カスタムフックでJS側から画面幅を検知し、Framer Motionのvariants値を切替（Tailwindの `md:` ではJS値を制御できないため）
- **useMediaQuery**: 既存の `hooks/` に存在しない場合、`window.matchMedia` ベースの軽量フックを新規作成

### 変更しないもの

- スマホレイアウト（768px未満）: 完全に現行維持
- Framer Motionのアニメーション基盤: 右パネルの切替に引き続き使用
- 3ステート構造のロジック: 変更なし
- テーマシステム: CSS変数による自動テーマ対応は維持
- SVG viewBox値: 290×290のまま、外側コンテナでサイズ制御

## テスト方針

- **既存テスト**: 破壊しないことを確認（`npm run test:run`）
- **ビルド確認**: `npm run build` 通過
- **視覚確認**: Playwright MCPでiPad横画面サイズ（1024×768）のスクリーンショットを撮影し、3ステートそれぞれのレイアウトを確認
