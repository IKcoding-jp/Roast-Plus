# 要件定義 — Issue #274 テーマプレビューカード個性強化 v2

## 概要

PR #273（#272）でリデザインされたテーマ設定画面について、3つの未解決課題を修正する。

## 背景・課題

### 課題1: アニメーションが控えめすぎる
- 炎アニメーション（dark-roast）: 最大透明度20%でほぼ見えない
- 葉アニメーション（matcha）: 固定12%透明度で動いても気づかない
- グロー（caramel）: 色強度13%でほぼ無効
- → ユーザーには「個性がない」と感じられる

### 課題2: ダーク系テーマの背景が似たり寄ったり
- default、dark-roast、matcha、caramel、christmas、dark の6テーマのうち5つが非常に暗い単色背景
- 一覧で並べると「全部暗い」状態

### 課題3: LIGHT/DARKバッジとアイコンの重なり
- カード上部で `flex items-start justify-between` により同一行に並んでいる
- 画面サイズによっては重なりが発生

## 要件

### 機能要件

| # | 要件 | 優先度 |
|---|------|--------|
| F1 | 各テーマカードにテーマ固有の背景グラデーションを適用する | 必須 |
| F2 | 各テーマカードに大型装飾アイコン（size=90, opacity=0.10）を右下背景に追加する | 必須 |
| F3 | 全アニメーション種別の強度を大幅に強化する（詳細は design.md 参照） | 必須 |
| F4 | LIGHT/DARKバッジをカード下部バー（スウォッチ行）の左端に移動する | 必須 |

### 非機能要件

| # | 要件 |
|---|------|
| N1 | `prefers-reduced-motion` 設定時はアニメーションを非表示にする（既存動作維持） |
| N2 | アクセシビリティ属性（aria-pressed, aria-hidden）を維持する |
| N3 | 既存テスト（ThemeSelector.test.tsx, theme.test.ts）を全件パスさせる |
| N4 | lint / build / test:run がゼロエラーで通過する |

## 受け入れ基準

- [ ] 7テーマのカードが並んだとき、各テーマの個性が一目でわかる
- [ ] ダーク系テーマの背景が互いに明確に異なる
- [ ] アニメーションが肉眼で確認できる強度になっている
- [ ] LIGHT/DARKバッジがアイコンと重ならない
- [ ] lint / build / test:run がゼロエラー

## スコープ外

- CSS変数・テーマ切替ロジック（ThemeProvider）の変更
- 設定画面以外のページへの影響
- 新テーマの追加
