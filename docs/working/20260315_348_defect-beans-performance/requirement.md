# 要件定義

**Issue**: #348
**作成日**: 2026-03-15
**ラベル**: performance

## ユーザーストーリー

ユーザー「コーヒー豆図鑑を開いたけど、豆が表示されるまで時間がかかってストレスを感じる」
アプリ「キャッシュとプログレッシブ表示で、再訪問時は即座に、初回でもスケルトン表示で待ち時間を感じさせない」

## 要件一覧

### 必須要件
- [ ] マスターデータのキャッシュ導入（再訪問時の即時表示）
- [ ] ローディングUX改善（全画面Loading → スケルトンカード表示）
- [ ] 画像読み込みの最適化（遅延読み込み、プレースホルダー）

### オプション要件
- [ ] Firestoreクエリの最適化（orderByサーバーサイドソート）
- [ ] 画像アップロード時の圧縮処理

## 非機能要件
- パフォーマンス: 再訪問時はキャッシュから200ms以内にカード表示開始
- パフォーマンス: 初回訪問時はスケルトン表示を100ms以内に開始
- モバイル対応: 低速ネットワークでもスケルトンで待ち時間を体感軽減

## 受け入れ基準
- [ ] 再訪問時、ページ遷移後にスケルトンではなくキャッシュデータで即座にカードが表示される
- [ ] 初回訪問時、全画面Loadingではなくスケルトンカードが表示される
- [ ] 画像は画面外のものが遅延読み込みされる
- [ ] 既存の検索・フィルター・ソート機能が正常に動作する
- [ ] `npm run build && npm run test:run` が通る
