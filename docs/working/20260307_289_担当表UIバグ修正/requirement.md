# requirement.md — #289 担当表UIの微調整

## 概要

担当表の `TableModals.tsx` / `DesktopTableView.tsx` にある3つのUIバグを修正する。

## バグ詳細

### Bug 1: 左右ラベル列の幅が視覚的に異なる

**現象**: 左ラベル列と右ラベル列に同じpx幅を設定しても、表示上わずかに幅が異なって見える。

**原因**:
1. 左ラベルセル (`DesktopTableView.tsx:266`) に `border-r` (1px) が付いており、右ラベルセルにはない。CSS box-sizing では border が内側に描画されるため、左の実コンテンツ幅が 1px 狭くなる
2. 新規追加行 (l.350-394) の右側 input は Add ボタンと flex スペースを共有しており、左の input より見た目の幅が短い

**修正方針**:
- 左右ラベルセルの内部パディングを揃える
- 新規追加行の左右 input のレイアウトを均一化する

### Bug 2: 高さ設定モーダルの「キャンセル」が2行になる

**現象**: 高さ設定モーダル（行ラベル設定モーダル）を開くと、削除・キャンセル・保存の3ボタンが横並びになるが「キャンセル」だけ2行に折り返される。

**原因**:
- モーダル幅: `max-w-xs` = 320px、内側パディング `p-6` = 48px → 実幅 272px
- 3ボタン + `gap-2`×2(16px) → 各ボタン幅 ≈ 85px
- Button の `px-4` (内側パディング 32px) を引いた残り ≈ 53px
- 「キャンセル」(5文字 ≈ 70px) が 53px に収まらず折り返す

**修正方針**:
- 高さ設定モーダルのボタン行のレイアウトを変更する
  - 案A: モーダル幅を `max-w-sm` (384px) に変更
  - 案B: 3ボタン構成を変え、削除を上部に配置して下段を2ボタン（キャンセル・保存）にする ← 推奨
  - 案C: ボタンに `whitespace-nowrap` を追加（サイズを絞る）

### Bug 3: 削除ボタンの背景が赤にならない

**現象**: 高さ設定モーダルとチーム編集モーダルの削除ボタンが赤くない。アイコンも赤色でない。

**原因**:
- `variant="danger"` → `bg-danger text-white hover:bg-danger/90`
- しかし `className` に `!bg-ground` を指定しており、`!important` で `bg-danger` が上書きされる
- 結果として `bg-ground`（明るい背景色）が適用され、赤背景が表示されない

**修正方針**:
- `TableModals.tsx` の2箇所の削除ボタンから `!bg-ground` を削除する
- `hover:!bg-red-50` も削除し、`variant="danger"` のホバースタイルに任せる
- flexレイアウト用の `!flex !items-center !justify-center !gap-1` は維持する
