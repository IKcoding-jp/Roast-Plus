# requirement.md — Issue #280

## 概要

コーヒー豆図鑑（`app/defect-beans`）でヘッダー統一後に上部UIが2行構成になり、iPad画面でフィルターとヘッダーが競合する問題を解消する。

## 背景

- `FloatingNav` ヘッダーを全ページ統一した結果、直下に `SearchFilterSection`（検索バー＋フィルターボタン）が配置され視覚的な競合が発生
- iPad画面での使用頻度が高く、上部スペースの節約が優先課題

## 要件

### 必須要件

| # | 要件 | 受け入れ基準 |
|---|------|-------------|
| 1 | ヘッダーの「ソート」ボタンを「フィルター」ボタンに変更 | ボタン名が「フィルター」になっている |
| 2 | 「フィルター」ボタン押下でモーダルが開く | モーダル内に検索・絞り込み・ソートの3セクションが表示される |
| 3 | モーダル内に検索フィールドを移動 | 「名称や特徴で検索...」のInputがモーダル内に表示される |
| 4 | モーダル内に絞り込みオプションを移動 | 「全て/省く/省かない」の選択肢がモーダル内に表示される |
| 5 | モーダル内にソートオプションを維持 | 「デフォルト/新しい順/古い順/名前昇順/名前降順」がモーダル内に表示される |
| 6 | `SearchFilterSection` をページから削除 | 検索バーとフィルターボタンの独立行が消える |

### 非機能要件

- 既存のフィルタリング・検索・ソートロジック（`filteredDefectBeans` useMemo）は変更しない
- `@/components/ui` の共通コンポーネント（`Modal` または `Dialog`）を使用してモーダルを実装
- テーマ対応は既存の CSS 変数で自動対応（変更不要）

## スコープ外

- フィルタリングロジックの変更
- ソートオプションの追加・削除
- 他ページへの影響

## 受け入れ基準

- [ ] iPad幅でヘッダーとコンテンツが競合しない
- [ ] 検索・絞り込み・ソートがすべてモーダル内で動作する
- [ ] モーダルを閉じても選択状態が維持される
- [ ] `npm run lint && npm run build` が通る
