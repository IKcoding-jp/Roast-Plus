# 要件定義: テーマセレクターのビジュアルリデザイン

**Issue**: #272
**タイプ**: feat
**規模**: 大
**作成日**: 2026-02-23

---

## 背景

現在の `ThemeSelector.tsx` は抽象的な斜め色帯（bg/surface/accent/text の4色）でテーマを表現している。色の組み合わせだけでは「このテーマはどんな雰囲気か」がパッと見で伝わりにくい。

## ユーザーストーリー

> 「テーマ選択画面を開いたとき、それぞれのテーマがどんな雰囲気なのかを色だけでなく文字・アイコン・アニメーションで直感的に感じ取れる」

## 機能要件

### FR1: フルイマーシブカード

- [ ] カード全体がテーマの `previewColors.bg` 色で塗られる
- [ ] 文字・アイコン色はすべて `previewColors.text` を使用する
- [ ] カード上部: テーマ専用アイコン（24px）+ LIGHT/DARK バッジ
- [ ] カード中部: テーマ名を大きなタイポグラフィで表示（text-2xl）
- [ ] カード中部下: 説明文（text-xs、opacity-75）
- [ ] カード下部: 色スウォッチ（3色ドット）+ 選択チェックアイコン

### FR2: タイポグラフィ表現

- [ ] プレビュー内のみ各テーマのfont-weight/letter-spacingを変える
- [ ] アプリ全体のフォントは変更しない
- [ ] テーマIDに対応した `fontStyle` メタデータを `lib/theme.ts` に追加

### FR3: テーマ専用アイコン

- [ ] 各テーマに1つの専用アイコンを割り当てる（react-icons）
- [ ] 絵文字は使用しない（シンプルなSVGアイコン）
- [ ] アイコンは `lib/theme.ts` の `ThemePreset` にメタデータとして持つ

### FR4: アンビエントアニメーション（常時ループ）

- [ ] 各テーマカードにテーマの雰囲気に合ったアニメーションを実装する
- [ ] アニメーションは常時ループ（ページ表示中ずっと動き続ける）
- [ ] Framer Motion `animate` + `repeat: Infinity` を使用
- [ ] 軽量なdivベース（canvasは使わない）

#### アニメーション仕様

| テーマ | アニメーション |
|--------|-------------|
| デフォルト | 湯気が静かに上昇（2〜3本の細い線が上へフェードアウト） |
| ダークロースト | 炎アイコンが scale 1.0→1.05 + opacity 0.8→1.0 でゆらぐ |
| ライトロースト | 光の粒子（小さい白丸）が下から上へゆっくり上昇・消滅 |
| 抹茶ラテ | 葉アイコンが rotate -3deg→3deg でゆっくり左右に揺れる |
| キャラメルマキアート | 背景に radial-gradient オーバーレイが pulse する |
| クリスマス | 白い雪粒（5〜8粒）がランダム速度でゆっくり落下 |
| ダークモード | 複数の小点が staggered タイミングで opacity 1→0.2→1 点滅 |

### FR5: 選択状態の視覚表現

- [ ] 非選択時: `border-2 border-transparent`
- [ ] ホバー時: `border-2 border-text/20`
- [ ] 選択時: `border-2 border-white/40 ring-2 ring-white/20`

## 非機能要件

- アニメーションのパフォーマンス: `will-change: transform` を活用、60fps維持
- `prefers-reduced-motion` メディアクエリを尊重（アニメーション無効化）
- 2カラムグリッドレイアウト維持

## 受け入れ基準

1. 7テーマすべてのカードがフルイマーシブスタイルで表示される
2. 各テーマのアイコンが適切に表示される
3. アンビエントアニメーションが常時ループで動く
4. 選択・非選択・ホバーの状態遷移が適切に機能する
5. `prefers-reduced-motion` でアニメーションが停止する
6. 既存テストが通過する
7. lint / build / test が全て通る

## スコープ外

- アプリ全体のフォント切り替え機能
- テーマ数の追加・削除
- テーマの並び順変更
