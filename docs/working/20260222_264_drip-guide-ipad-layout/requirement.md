# 要件定義 — Issue #264

## 概要

ドリップガイド実行画面（`/drip-guide/run`）のiPadランドスケープ向けレイアウト最適化。

## ユーザーストーリー

コーヒーショップのスタッフが、iPadをカウンターに置いてドリップ作業をする際に：
- **全ステップの工程**（蒸らし→2投目→…）を左サイドバーで把握しながら
- **現在の注水目標・カウントダウン**を右カラムで大きく確認できる

## 採用デザイン

**B-1: タイマーヘッダー統合型**（Design Labで承認済み、`IpadLayoutB1.tsx` 参照）

```
+──────────────────────────────────────────────────────────+
│ ← │ 04:30  [████████░░░░]  4:6メソッド  │ Step 2 / 5  │
+─────────────────────┬────────────────────────────────────+
│ ✓ 蒸らし            │                                    │
│   00:00・30g        │  [Drop] 2投目（味：40%）           │
│ ▶ 2投目             │         60g まで注ぐ               │
│   00:45・60g        │  中心から外へ円を描くように注ぐ     │
│ ○ 濃度調整          │  ────────────────────────────      │
│ ○ 仕上げ①②        │  → 濃度調整   あと 35秒           │
+─────────────────────┴────────────────────────────────────+
│                    [↺]  [▶]  [×]                        │
+──────────────────────────────────────────────────────────+
```

## ブレイクポイント

| 幅 | レイアウト |
|---|---|
| < 768px (モバイル) | 既存の縦積みレイアウト（**変更なし**） |
| ≥ 768px (iPadランドスケープ) | B-1 2カラムレイアウト |

## 受け入れ基準

- [ ] iPad (768px+) で左サイドバー（全ステップリスト）が表示される
- [ ] 左サイドバーに各ステップの開始時刻・注水量・完了状態が表示される
- [ ] iPad で RunnerHeader にタイマーと全体進捗バーが表示される
- [ ] iPad で TimerDisplay が非表示になる（タイマーはヘッダーに移動）
- [ ] iPad で StepInfo が `max-w-md` 制限なしに横幅を使える
- [ ] iPad で フッターボタンがやや小さくなる（play: w-16、サブ: size=20）
- [ ] モバイル（< 768px）では既存のレイアウトが完全に維持される
- [ ] マニュアルモードでも iPad レイアウトが正常動作する
- [ ] 全既存テストがパスする
- [ ] ビルド・Lintエラーなし

## 非対応スコープ

- ポートレート（縦向き）の最適化は対象外
- 1024px超（デスクトップ）の追加最適化は対象外
- アニメーション追加は対象外（既存のframer-motionをそのまま活用）
