# 要件定義

**Issue**: #232
**作成日**: 2026-02-13
**ラベル**: enhancement

## ユーザーストーリー
ユーザー「ホーム画面のヘッダーがしっくりこない。いくつかのデザイン案を実際の画面で見比べて、一番いいのを選びたい」
アプリ「開発者モードで複数のヘッダーデザインをリアルタイムに切り替えて比較できるデバッグUIを提供する」

## 要件一覧

### 必須要件
- [ ] 現在のヘッダー（`app/page.tsx:297-376`）をコンポーネントとして抽出
- [ ] 3〜5個のヘッダーデザインバリアントを作成
- [ ] 開発者モード時にフローティング切替UIを表示
- [ ] バリアント切替でヘッダーがリアルタイムに差し替わる
- [ ] 全5テーマ（default, christmas, dark-roast, light-roast, vintage-coffee）で動作

### オプション要件
- [ ] 切替UI上にバリアント名・サムネイルを表示
- [ ] 「採用」ボタンで選択を確定（ログ出力 or localStorage保存）

## 受け入れ基準
- [ ] 開発者モードON時のみ切替UIが表示される
- [ ] バリアント切替が即座に反映される（リロード不要）
- [ ] 全テーマで各バリアントが正しく表示される
- [ ] デバッグ機能は一時的であり、採用決定後に容易に削除できる構造
- [ ] 非開発者モードではデバッグ機能のコードが表示に影響しない

## 備考
- デザインバリアントは `/frontend-design` スキルで生成
- 最終的に採用されたデザインを本実装し、デバッグコードは削除する
