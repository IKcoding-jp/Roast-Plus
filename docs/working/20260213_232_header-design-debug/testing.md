# テスト計画

## テスト戦略

一時的なデバッグ機能のため、テストは最小限にとどめる。
ヘッダー抽出後の既存動作の保証と、デバッグUIの基本動作をカバーする。

### ユニットテスト

#### HomeHeader抽出の動作保証
- `components/home/__tests__/HomeHeader.test.tsx`
  - 通常モードでロゴ（"Roast" + "Plus"）が表示される
  - 時計ボタンが表示される
  - 開発者モードON時にデバッグボタンが表示される

#### HeaderDebugSwitcher
- `components/home/__tests__/HeaderDebugSwitcher.test.tsx`
  - バリアントボタンが全て表示される
  - ボタンクリックで `onVariantChange` コールバックが呼ばれる
  - 現在の選択がハイライトされる

### 手動テスト（重要）
- [ ] 全5テーマ × 全バリアントの表示確認
- [ ] レスポンシブ（スマホ/タブレット）での表示確認
- [ ] 開発者モードON/OFFでデバッグUIの表示/非表示
- [ ] バリアント切替がリアルタイムに反映される

## カバレッジ目標
- 本機能は一時的なため、厳密なカバレッジ目標は設定しない
- 抽出した `HomeHeader` の基本レンダリングテストのみ必須
