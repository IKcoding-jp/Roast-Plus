# 並列UIデザイン探索スキル

UIデザインの失敗パターン（「何度も試してリバート」）を根本から解消する。
実装前に3方向の仕様書を同時生成し、ユーザーが選択してから1度で実装する。

## ワークフロー概要

```
対象確認 → 3エージェント並列起動 → 仕様書×3 → ユーザー選択 → /frontend-design で実装
```

---

## Step 1: 対象コンポーネントの把握

実装対象のUIコンポーネント/ページを読み込み、以下を把握する:

- 現在のファイルパス・コンポーネント構造
- 使用中のCSS変数・共通コンポーネント
- 変更したい箇所の具体的な要件

---

## Step 2: 3方向の仕様書を並列生成

**Taskツールで以下の3エージェントを同時起動する:**

### Agent A: Minimal（ミニマル）
```
対象ファイルを読み込み、以下の設計方向性でテキスト仕様書を作成せよ。
コードは書かない。仕様書のみ出力。

方向性: Minimal
- 余白を最大化（padding: 24-32px）
- カラーはセマンティックトークンのみ（bg-page, text-ink, border-edge）
- ボーダー: 細い（border-edge, 1px）
- タイポグラフィ: 細め（font-normal）
- アイコン: 最小限

出力形式:
## レイアウト構造（テキスト図）
## 使用コンポーネント（@/components/uiから）
## サイズ値（具体的なpx/remで）
## 色（CSS変数名で）
## スペーシング（具体的なpxで）
## 変更の要点（3行以内）
```

### Agent B: Bold（ボールド）
```
対象ファイルを読み込み、以下の設計方向性でテキスト仕様書を作成せよ。
コードは書かない。仕様書のみ出力。

方向性: Bold
- アクセントカラーを積極活用（bg-spot, text-spot）
- 角丸を強調（rounded-2xl以上）
- タイポグラフィ: 大きめ・太め（text-xl+, font-semibold）
- カードにドロップシャドウ（shadow-md）
- ボタン: primary variantを基本

出力形式:
## レイアウト構造（テキスト図）
## 使用コンポーネント（@/components/uiから）
## サイズ値（具体的なpx/remで）
## 色（CSS変数名で）
## スペーシング（具体的なpxで）
## 変更の要点（3行以内）
```

### Agent C: Playful（プレイフル）
```
対象ファイルを読み込み、以下の設計方向性でテキスト仕様書を作成せよ。
コードは書かない。仕様書のみ出力。

方向性: Playful（RoastPlusのコーヒーテーマに合わせた遊び心）
- コーヒー関連の装飾要素（coffee variant、アイコン活用）
- カラフルなBadgeの積極利用
- Framer Motionアニメーション（軽微なもの）
- インタラクティブな要素（hover効果を強調）
- XP・ストリーク等のゲーミフィケーション要素を活かす

出力形式:
## レイアウト構造（テキスト図）
## 使用コンポーネント（@/components/uiから）
## サイズ値（具体的なpx/remで）
## 色（CSS変数名で）
## スペーシング（具体的なpxで）
## 変更の要点（3行以内）
```

---

## Step 3: 仕様書の比較提示

3エージェントの出力が揃ったら、以下の形式でユーザーに提示する:

```
## UIデザイン探索結果

### Option A: Minimal
[Agent Aの仕様書]

---

### Option B: Bold
[Agent Bの仕様書]

---

### Option C: Playful
[Agent Cの仕様書]

---

どの方向性で実装しますか？
（A/B/C から選択、または「AとBを混ぜて」のような指示も可）
```

---

## Step 4: 選択方向で実装

ユーザーが選択したら、選択された仕様書に従って実装する。

**実装は `/frontend-design` スキルと組み合わせて実行することを推奨。**

```
/frontend-design を使用してOption [選択]の仕様書に従い実装してください。
```

### 実装時のルール（CLAUDE.mdのUI必須ルールに従う）

- 共通コンポーネント（`@/components/ui`）を使用
- ハードコード色（`bg-white`等）禁止 → CSS変数を使用
- 仕様書に記載のない変更は加えない
- 実装後は `/dev/design-lab` または Playwright MCPでスクリーンショット確認

---

## 使用例

```
/parallel-ui-design

コーヒークイズの問題カード（app/coffee-trivia/）のデザインを
3方向で探索してください。
要件: 正解/不正解の状態がより直感的にわかるようにしたい。
```

---

## ⚠️ 注意事項

- **このスキルはUIデザインの仕様作成まで行う。実装は `/frontend-design` に委ねる。**
- 仕様書はコードではなくテキスト図と変数名で記述（実装前の選択をサポートするため）
- ユーザーの承認なしに実装フェーズへ進まない
