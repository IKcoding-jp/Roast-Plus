# 要件定義

**Issue**: #307
**作成日**: 2026-03-07
**ラベル**: enhancement

## ユーザーストーリー

ユーザー「シャッフルで毎回同じ作業になるのが気になる。組み合わせより担当の方を変えてほしい」
アプリ「詳細設定で『同じ作業の連続を避ける』を選ぶと、担当の変更を優先してシャッフルします」

## 要件一覧

### 必須要件
- [ ] 詳細設定モーダルに「シャッフルの優先順位」ラジオボタンを表示する
- [ ] 「同じ人との組み合わせを避ける」（ペア優先）を選択できる
- [ ] 「同じ作業の連続を避ける」（担当優先）を選択できる
- [ ] 選択した優先順位がFirestoreに保存され、次回以降も反映される
- [ ] シャッフル実行時に選択された優先順位に応じて制約レベル順序が切り替わる
- [ ] 既存ユーザーはデフォルト「ペア優先」で動作変化なし（後方互換）

### オプション要件
- なし（YAGNI: 最小限の2択に絞る）

## 非機能要件
- パフォーマンス: シャッフル計算時間に影響なし（制約レベル順序の変更のみ）
- アクセシビリティ: ラジオボタンに適切なaria-label
- モバイル対応: 既存モーダルのレスポンシブに準拠

## 受け入れ基準
- [ ] 詳細設定モーダルに2択のラジオボタンが表示される
- [ ] 各選択肢に説明テキスト（トレードオフ）が表示される
- [ ] 設定変更がFirestoreに即時保存される
- [ ] 「担当優先」設定でシャッフル実行時、行の連続回避が優先される
- [ ] 「ペア優先」設定でシャッフル実行時、従来と同じ動作になる
- [ ] 設定未保存（既存ユーザー）の場合、デフォルトで「ペア優先」が選択される
- [ ] lint / build / test すべてパス
