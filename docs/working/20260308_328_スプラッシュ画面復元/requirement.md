# requirement.md — #328 スプラッシュ画面復元

## 概要

Issue #234 で削除されたスプラッシュ画面コンポーネントを復元する。
PWAとして起動した際のカスタムアニメーションスプラッシュが表示されない問題を修正する。

## 背景・経緯

- `#234` にて「不要になったため削除」されたが、改めて復活させる要望が出た
- 現在はOSネイティブのPWAスプラッシュのみが表示されており、ブランドらしいカスタムアニメーションがない状態
- iOS / Android 両方で問題が確認されている

## 受け入れ基準

1. PWAとして起動した際（ホーム画面のアイコンをタップ）、カスタムスプラッシュアニメーションが表示される
2. セッション中は初回起動時のみ表示（2回目以降は表示しない）
3. 5パターン（FadeUp/ScaleBreathe/LetterStagger/SlideReveal/GlowPulse）からランダムに選択される
4. スプラッシュ表示後、スムーズにホーム画面に遷移する
5. lint / build / test がすべてパスする

## スコープ外

- OSネイティブスプラッシュの完全削除（技術的に不可能のため、視覚的シームレス化で代替）
- 開発用プレビューページ（`app/dev/splash-preview/`）の復元（不要）
- Lottieデバッグモーダルの復元（不要）
- 開発者用スプラッシュ再生ボタンの復元（不要）

## OSネイティブスプラッシュについて

manifest.json の `background_color: #261a14` = カスタムスプラッシュの背景色（#261a14）と同一。
シームレスな切り替えにより、OSネイティブスプラッシュの存在感を最小化する。
