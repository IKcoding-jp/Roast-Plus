# 要件定義

**Issue**: #330
**作成日**: 2026-03-11
**ラベル**: bug

## ユーザーストーリー

ユーザー「担当表でシャッフルボタンを押したら、一部のメンバーが表から消えて空欄になっている。全員が必ずどこかに割り当てられるようにしてほしい」
アプリ「班内シャッフル時に、全アクティブメンバーを確実にいずれかのスロットに配置する」

## 要件一覧

### 必須要件
- [ ] バックトラッキング探索時に、選択した組み合わせの班分布がスロット構成と一致することを検証する
- [ ] 全アクティブメンバーが必ず1つのスロットに配置される（memberId === null のスロットは、メンバー数 < スロット数の場合のみ許容）
- [ ] タスク除外設定・ペア除外設定・履歴制約との組み合わせでも正常に動作する
- [ ] 既存テストがすべてパスする（リグレッションなし）

### オプション要件
- [ ] 班間の人数差がある場合のエッジケースを明示的にテスト

## 非機能要件
- パフォーマンス: 8人規模で体感遅延なし（現状と同等）
- 後方互換性: crossTeamShuffle=true（班またぎ）の動作に影響なし

## 受け入れ基準
- [ ] 班内シャッフルで全アクティブメンバーがいずれかのスロットに配置される
- [ ] タスク除外設定ありでも、除外対象外のタスクには必ず配置される
- [ ] ペア除外設定ありでも、メンバーが消失しない
- [ ] 既存テスト（357件のshuffle関連）がすべてパス
- [ ] lint / build / test:run すべてパス
