## テスト戦略

### 手動テスト（必須）
ページ遷移時の視覚的な問題のため、手動での目視確認が主要な検証手段。

#### 確認ポイント
- 白フラッシュが発生しないこと
- Loading コンポーネントの背景がテーマと一致すること
- テーマ切替後のページ遷移でも問題ないこと

### 自動テスト
- ビルド: `npm run build` がエラーなく完了
- Lint: `npm run lint` がエラー・warningゼロ
- 既存テスト: `npm run test` が全件パス
