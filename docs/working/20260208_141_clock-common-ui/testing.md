# テスト計画: デジタル時計ページの共通UI化とテーマシステム対応

## テスト方針
リファクタリング（UI置換）が主体のため、既存テストの維持 + 必要に応じて新規テスト追加。

## 対象テストファイル

### 既存テスト確認
- `app/clock/` 関連のテストファイル有無を確認
- `components/clock/` 関連のテストファイル有無を確認

### 新規テスト（必要に応じて）
- ClockSettingsModal のレンダリングテスト
- Switch/Button/Modal の共通コンポーネントとの統合テスト

## テストケース

### 1. ClockSettingsModal
- モーダルの表示/非表示
- テーマ切替
- フォント切替
- トグルスイッチの操作（24時間表示、秒表示、日付表示）
- リセットボタン

### 2. page.tsx
- 時計表示の基本レンダリング
- 戻るリンクの存在確認
- 設定ボタンのクリックでモーダル表示

## カバレッジ目標
- 変更ファイルのカバレッジ維持（低下しないこと）
