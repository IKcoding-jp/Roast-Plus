# RoastPlus - Product Vision

**最終更新**: 2026-02-21

---

## プロダクト概要

RoastPlusは、コーヒー豆加工業務（焙煎・ハンドピック・ドリップ等）を行う現場向けの業務効率化PWAアプリである。
8名のチームが日常的に利用し、作業分担・進行管理・時間計測・知識習得・テイスティング評価を一つのアプリケーションで完結させる。

紙・Excel・口頭伝達に依存していた業務フローをデジタル化し、スマートフォン一つで現場の全業務を支援することを目指している。

---

## ターゲットユーザー

| 属性 | 詳細 |
|-----|------|
| **規模** | 小規模コーヒー加工現場（5〜10名規模） |
| **利用シーン** | 作業中にスマートフォンで操作するスタッフ |
| **IT習熟度** | 低〜中（直感的な操作が必須、専門的なIT知識を前提としない） |
| **利用頻度** | 毎日（業務中に常時利用） |
| **デバイス** | 主にスマートフォン（iOS/Android）、タブレット・PCでも利用可能 |

---

## コアバリュー

### 1. 現場で使われること

- **8名のリアルユーザーが毎日利用** している実績を持つ
- 机上の空論ではなく、実際の業務フローに密着した設計
- ユーザーフィードバックを迅速に反映し、現場の課題を即座に解決する
- 使われないアプリは価値がない。利用率と満足度が最重要指標

### 2. モバイルファースト

- **PWA対応**: ホーム画面追加でネイティブアプリのように起動、オフライン動作
- **スマートフォン最適化**: 片手操作、大きなタップ領域、作業手袋でも操作しやすいUI
- **レスポンシブデザイン**: タブレット・PCでも快適に利用可能
- **カスタムService Worker**: Network First戦略で常に最新データを優先

### 3. 属人化解消

- **作業知識のドキュメント化**: ドリップガイド（4:6メソッド等）、焙煎記録で暗黙知を形式知化
- **公平な担当割り当て**: 過去履歴を考慮した自動シャッフルで偏りを解消
- **新人オンボーディング**: コーヒークイズ（FSRS間隔反復学習）による体系的な知識習得
- **欠点豆図鑑**: ハンドピック作業の判断基準を写真付きで共有

### 4. データ駆動

- **作業時間の可視化**: 焙煎タイマー・ドリップタイマーによる計測と履歴記録
- **Firestoreリアルタイム同期**: クラウドに永続化し、全デバイスで即時共有
- **AI分析**: テイスティング評価のAI分析（OpenAI GPT-4o）、スケジュール画像のOCR読み取り
- **学習進捗の定量化**: XP・レベル・ストリークによるゲーミフィケーション

---

## ビジョン

**コーヒー加工現場における「紙・Excel・口頭」の業務を、スマートフォン一つで完結させる。**

属人化した知識をデジタル化し、チーム全員が同じ品質の作業を行える環境を実現する。
データに基づく改善サイクルを回し、コーヒー品質の継続的な向上を支援する。

## スコープ

### 含まれるもの

| カテゴリ | 機能 | 概要 |
|---------|------|------|
| **作業管理** | 担当表（Assignment） | 自動シャッフル、出欠管理、履歴記録 |
| **作業管理** | スケジュール（Schedule） | 画像OCRによるスケジュール読み取り |
| **焙煎支援** | 焙煎タイマー（Roast Timer） | 時間計測、フェーズ記録、温度ラベルOCR |
| **抽出支援** | ドリップガイド（Drip Guide） | レシピ管理、4:6メソッド、タイマー付きガイド |
| **品質評価** | テイスティング（Tasting） | フレーバーホイール評価、AI分析 |
| **知識習得** | コーヒークイズ（Coffee Quiz） | FSRS間隔反復学習、ゲーミフィケーション |
| **知識共有** | 欠点豆図鑑（Defect Beans） | 欠点豆の種類・写真・説明 |
| **システム** | 設定・テーマ（Settings） | 7テーマプリセット、開発者モード |

### 含まれないもの

| 項目 | 理由 |
|-----|------|
| **B2C向けコーヒー愛好家アプリ** | 焙煎業務に特化（業務用途のみ） |
| **複数店舗・複数テナント管理** | シングルテナント前提（1チーム向け） |
| **複雑な権限管理（RBAC等）** | Firestore Security Rulesで簡易実装 |
| **在庫管理システムとの連携** | 現時点ではスコープ外 |
| **EC・販売管理機能** | 業務支援に限定 |
| **外部SaaS連携（Slack, LINE等）** | 現時点ではスコープ外 |

## 技術アーキテクチャ概要

RoastPlusは**Static PWA + Firebase BaaS**のアーキテクチャを採用している。
サーバーサイドのAPI Routesは使用せず、静的エクスポート（`output: 'export'`）でビルドし、AI処理はFirebase Cloud Functionsに委譲する。

```
┌──────────────────────────────────────────────────────┐
│              クライアント（Static PWA）                │
│   Next.js 16 (App Router) / React 19.2.0 / TS 5     │
│   Tailwind CSS v4 / Framer Motion 12.x / Lottie     │
│   Custom Service Worker (Network First)              │
│   7テーマプリセット (next-themes + CSS変数)            │
│   状態管理: React useState のみ                       │
└──────────────┬───────────────────────────────────────┘
               │ Firebase SDK（クライアント直接接続）
               ↓
┌──────────────────────────────────────────────────────┐
│                Firebase（BaaS）                       │
│   - Authentication（Google, Email/Password）          │
│   - Firestore（NoSQL, リアルタイム同期, onSnapshot）   │
│   - Storage（画像アップロード）                        │
│   - Hosting（静的ファイル配信）                        │
└──────────────┬───────────────────────────────────────┘
               │ httpsCallable（クライアントから呼び出し）
               ↓
┌──────────────────────────────────────────────────────┐
│          Firebase Cloud Functions                     │
│   - ocrScheduleFromImage（スケジュール画像OCR）        │
│   - analyzeTastingSession（テイスティングAI分析）      │
│   - OPENAI_API_KEY: Firebase Secret Manager管理       │
└──────────────┬───────────────────────────────────────┘
               │ API呼び出し
               ↓
┌──────────────────────────────────────────────────────┐
│              OpenAI API                               │
│   - GPT-4o Vision（OCR処理）                          │
│   - GPT-4o（テイスティング分析テキスト生成）            │
└──────────────────────────────────────────────────────┘
```

### アーキテクチャの特徴

| 特徴 | 詳細 |
|-----|------|
| **静的エクスポート** | `output: 'export'` でビルド。サーバーサイドAPI Routesなし |
| **クライアント直接接続** | Firebase SDKでFirestore/Auth/Storageに直接アクセス |
| **サーバーレスAI** | AI処理はCloud Functions経由。クライアントにAPIキーを持たない |
| **カスタムService Worker** | `public/sw.js` に手書き実装。next-pwaは使用しない |
| **状態管理** | React useState + Firestoreリアルタイム同期。外部状態管理ライブラリ不使用 |
| **テーマシステム** | next-themes + CSS変数 + `data-theme`属性。7テーマプリセット |
| **デプロイ先** | Firebase Hosting / Vercel（デュアルデプロイ対応） |
