# テスト計画

## テスト戦略

### 手動テスト（メイン）

テーマ対応はビジュアル変更のため、手動での目視確認が中心。

#### 通常モード確認
- [ ] `/terms` - 利用規約ページの表示が現状と同等
- [ ] `/privacy-policy` - プライバシーポリシーページの表示が現状と同等
- [ ] `/contact` - お問い合わせページの表示が現状と同等
- [ ] `/contact` - フォーム送信・エラー表示が正常
- [ ] `/consent` - 同意画面の表示が現状と同等
- [ ] `/consent` - チェックボックス操作・ボタン有効/無効が正常

#### クリスマスモード確認
- [ ] `/terms` - テキストが読める、背景が適切
- [ ] `/privacy-policy` - テキストが読める、背景が適切
- [ ] `/contact` - フォーム入力が使える、エラー表示が見える
- [ ] `/consent` - チェックボックスが見える、ボタンが適切な色

### 自動テスト

- [ ] 既存テストの pass 確認（テーマ変更によるリグレッションなし）
- [ ] Lint チェック pass

## カバレッジ目標

- 既存テストの維持（新規テスト追加は不要、ビジュアル変更のため）
