# 要件定義

**Issue**: #304
**作成日**: 2026-03-07
**ラベル**: enhancement

## ユーザーストーリー

ユーザー「シャッフルするとき、普段は班の中だけで担当を入れ替えたい。でも班をまたいで混ぜたいときもあるから、設定で切り替えられるようにしてほしい」
アプリ「詳細設定モーダルの『班をまたいでシャッフル』トグルをONにすると、班を超えたシャッフルが有効になります。OFFの場合は班内のタスク入れ替えのみ行います」

## 要件一覧

### 必須要件
- [ ] 統合詳細設定モーダルを実装（セクション分割式）
  - セクション1: シャッフル設定（全ユーザーに表示）
  - セクション2: ペア除外設定（isDeveloperModeのみ表示）
- [ ] 「班をまたいでシャッフル」トグルを追加（デフォルト: OFF）
- [ ] トグルOFF時: メンバーは自身の `teamId` に一致するスロットのみに配置
- [ ] トグルON時: 現在の動作を維持（班を自由にまたいで配置）
- [ ] 設定をFirestore `assignmentSettings/shuffle` に永続化
- [ ] 設定ボタン（HiCog）を全ユーザーに常時表示（isDeveloperMode制限を解除）

### オプション要件
- [ ] なし（YAGNI）

## 非機能要件
- パフォーマンス: シャッフル計算時間に影響なし（班内制約は候補フィルタの追加のみ）
- アクセシビリティ: トグルにaria-label付与
- モバイル対応: 既存モーダルのレスポンシブ構造を踏襲

## 受け入れ基準
- [ ] 設定ボタンが全ユーザーに表示される
- [ ] 「班をまたいでシャッフル」トグルがOFF時、シャッフル結果で全メンバーが元の班に留まる
- [ ] トグルON時、現在の動作と同等（班をまたいだ配置が可能）
- [ ] 設定がFirestoreに保存され、リロード後も維持される
- [ ] ペア除外設定セクションはisDeveloperModeのみ表示
- [ ] lint / build / test 全パス
