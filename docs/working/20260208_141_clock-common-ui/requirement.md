# 要件定義: デジタル時計ページの共通UI化とテーマシステム対応

## Issue
- **番号**: #141
- **タイトル**: refactor(clock): デジタル時計ページの共通UI化とテーマシステム対応

## 背景
デジタル時計ページ（`app/clock/`）は共通UIコンポーネントを使用しておらず、独自のボタンやモーダルを実装している。共通UI化プロジェクトの一環として、これらを `@/components/ui` の共通コンポーネントに置換する。

## 特殊事情
時計ページは**独自のテーマシステム**（5つのカラーテーマ: light/dark/coffee/green/lightblue）を持っている。これはアプリ全体の`data-theme`ベースのテーマ（通常/クリスマス）とは独立したもの。時計のテーマは時計表示の見た目（背景色、文字色、アクセント色）を制御するためのもので、アプリ全体のテーマとは目的が異なる。

## 要件

### 1. 共通UIコンポーネント化
- [ ] `page.tsx` の独自`<button>`（設定ボタン）→ `<IconButton>` に置換
- [ ] `page.tsx` の独自`<Link>`（戻るリンク）→ `<BackLink>` に置換
- [ ] `ClockSettingsModal.tsx` の独自モーダル → `<Modal>` に置換
- [ ] `ClockSettingsModal.tsx` の独自閉じるボタン → `<IconButton>` に置換
- [ ] `ClockSettingsModal.tsx` の独自トグルスイッチ → `<Switch>` に置換
- [ ] `ClockSettingsModal.tsx` のリセットボタン → `<Button>` に置換

### 2. テーマ対応（アプリ全体のテーマ）
- [ ] ページ背景 → CSS変数 `bg-ground` ベース + 時計テーマによるオーバーライド
- [ ] モーダル背景 → `bg-overlay` を使用
- [ ] テキスト色 → `text-ink`, `text-ink-sub` をベースに
- [ ] `useChristmasMode` フック導入（アプリ全体のテーマ対応）

### 3. 検証
- [ ] lint / build / test 通過
- [ ] 通常モード・クリスマスモード両方で動作確認

## 受け入れ基準
1. 独自UIが共通コンポーネントに置換されている
2. アプリ全体のテーマ（通常/クリスマス）に対応している
3. 時計固有のテーマ（5色）は引き続き正常動作する
4. 既存の時計設定（localStorage保存）との互換性が保たれる
